===SECTION: 프로젝트 개요===
프로젝트명: 팀메모리 (TeamMemory)
목적: 팀의 회의록, 이메일, 채팅 등 업무 자료를 AI로 분석해 Todo, 결정사항, 이슈를 자동 추출하고 대시보드로 시각화하는 서비스
기간: 2026년 5월 7일 ~ 2026년 6월 28일 (8주)
팀원: 김희진(팀장), 이성우(AI), 김예은(인프라), 김성호(백엔드), 박주원(프론트)
===SECTION_END===

===SECTION: 기술 스택===
백엔드: Python 3.11, FastAPI, MySQL 8.0
AI/ML: LangChain, FAISS (벡터DB), Azure OpenAI (gpt-4o, text-embedding-3-large)
인프라: Azure VM (Ubuntu 22.04, B2s 플랜), GitHub Actions
프론트엔드: React (Vite), Tailwind CSS
기타: python-dotenv, pydantic, uvicorn, faiss-cpu, numpy

벡터DB 선택 이유:
- ChromaDB 제외: Windows에서 onnxruntime DLL 충돌 오류 발생
- FAISS 채택: faiss-cpu 패키지, 로컬 파일 저장, MySQL과 독립적으로 동작
- faiss_db/ 폴더는 .gitignore 등록 - 팀원 각자 로컬 생성 필요
- 추후 서비스 규모 확장 시 Azure AI Search 전환 고려
===SECTION_END===

===SECTION: 아키텍처===
전체 흐름:
1. 사용자가 문서 업로드 (회의록, 이메일, 채팅 로그 등)
2. FastAPI가 파일 수신 후 BackgroundTasks로 비동기 처리 시작
3. chunker.py: 문서 유형별 청킹 (회의 단위, 이메일 단위, 이슈 단위 등)
4. embedder.py: Azure OpenAI text-embedding-3-large로 벡터화 후 FAISS 저장
5. summarizer.py: gpt-4o로 Todo/결정사항/이슈 JSON 추출
6. MySQL에 결과 저장 (todos, issues, ai_summaries 테이블)
7. React 대시보드에서 결과 표시

RAG 챗봇 흐름:
1. 사용자 질문 입력
2. retriever.py: FAISS에서 유사 청크 top-3 검색
3. summarizer.py: 검색 결과 + 질문 → gpt-4o → 답변 생성
4. 채팅 UI에 답변 표시

데이터 저장 구조:
- MySQL: 구조화 데이터 (users, documents, todos, issues, ai_summaries)
- FAISS: 벡터 데이터 (faiss_db/index.faiss, texts.pkl, metadatas.pkl)
===SECTION_END===

===SECTION: 환경 설정===
1. 레포 클론: git clone https://github.com/team/team-memory.git
2. 가상환경 생성: python -m venv venv && venv\Scripts\activate (Windows)
3. 패키지 설치: pip install -r requirements.txt
4. 환경변수 설정: .env.example을 .env로 복사 후 값 입력
   - AZURE_OPENAI_API_KEY: 이성우에게 DM 요청
   - AZURE_OPENAI_ENDPOINT: 이성우에게 DM 요청
   - AZURE_OPENAI_API_VERSION: 2024-02-01
   - AZURE_OPENAI_EMBEDDING_DEPLOYMENT: text-embedding-3-large
   - AZURE_OPENAI_CHAT_DEPLOYMENT: gpt-4o
   - FAISS_DIR: ./faiss_db
   - DB_PASSWORD: 김성호에게 DM 요청
5. FAISS DB 생성 (최초 1회 필수): python -m app.ai.embedder
6. 서버 실행: uvicorn app.main:app --reload

주의: faiss_db/는 gitignore 처리됨. 클론 후 반드시 5번 실행할 것
===SECTION_END===
