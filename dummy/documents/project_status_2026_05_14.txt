개발 규칙:
- 브랜치: main(배포용) / dev(통합) / feat/이름(개발)
- PR: dev로 올리고 팀원 1명 리뷰 후 머지
- 커밋: feat/fix/docs/refactor/test 접두사 사용
- 매주 금요일 dev → main 머지
- 소통 채널: 디스코드 (#개발 #이슈 #공지)

주간 루틴:
- 월요일: 주간 목표 합의 (15분)
- 수요일: 중간 점검 (15분)
- 금요일: 완료 확인 + dev → main 머지 (30분)

환경 설정 노하우:
- API 키: 이성우에게 디스코드 DM 요청 (AZURE_OPENAI_API_KEY, AZURE_OPENAI_ENDPOINT)
- DB 비밀번호: 김성호에게 디스코드 DM 요청
- FAISS DB: 클론 후 python -m app.ai.embedder 반드시 실행 (faiss_db/ gitignore 처리됨)
- Windows 개발 환경: Git Bash 사용 권장 (cp 명령어 호환)
- 인코딩: 한국어 파일 cp949 오류 시 chunker.py가 자동 재시도

기술 결정 배경:
- ChromaDB → FAISS 전환: Windows onnxruntime DLL 충돌로 인해 faiss-cpu 채택
- FastAPI BackgroundTasks 선택: Celery는 Redis 추가 필요, 8주 규모엔 BackgroundTasks 충분
- gpt-4o 선택: 50만원 예산 내 충분히 사용 가능, gpt-4o-mini 대비 한국어 품질 우수
- FAISS 향후 전환: 서비스 확장 시 Azure AI Search 검토 예정

파일 명명 규칙:
- 회의록: meeting_YYYY_MM_DD_주제.txt
- 보고서: report_YYYY_MM_DD_작성자.txt
- 이메일: email_YYYY_MM_DD.txt
- 채팅: chat_YYYY_MM_DD.txt
- CSV: tasks_YYYY_MM_period.csv
