# 아키텍처 결정 기록

## ADR-001: Harness를 저장소 루트에 적용

상태: 승인됨

`dev` 브랜치의 애플리케이션은 이미 `opsradar2/`에 있습니다. Harness 제어
파일은 저장소 루트에 추가합니다. 이렇게 해야 phase 실행이 복사된 템플릿
디렉터리가 아니라 실제 애플리케이션 트리를 수정하고 검증합니다.

## ADR-002: 기존 FastAPI 계층 구조 유지

상태: 승인됨

현재 프로젝트에는 엔드포인트, 스키마, 서비스, 리포지토리, 모델 디렉터리가
이미 존재합니다. 새 구현 작업은 이 분리를 유지해야 하며, 임시 구현
엔드포인트를 완성할 때 영속성 처리를 라우트 핸들러 밖으로 이동해야 합니다.

## ADR-003: 기능 완성 전에 계약 안정화

상태: 승인됨

`dev`에는 프론트엔드와 백엔드가 모두 존재하지만 여러 엔드포인트 모듈이
아직 미완성입니다. 첫 번째 Harness phase는 더 넓은 구현에 들어가기 전에
라우트 계약을 감사하고 검증을 도입합니다. 이를 통해 데이터 모양이
실수로 바뀌는 병합 사고를 줄입니다.
