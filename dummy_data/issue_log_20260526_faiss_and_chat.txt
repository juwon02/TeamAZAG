이슈 로그 — FAISS 인덱스 유실 위험
================================================================
이슈 ID: ISSUE-006
제목: 서버 재시작 시 FAISS 인덱스 파일 유실 가능성
심각도: Medium
상태: Resolved
발생일: 2026-05-26
해결일: 2026-05-28
담당자: 이성우, 김예은
보고자: 이성우

----------------------------------------------------------------
1. 증상
----------------------------------------------------------------
- Azure VM 재시작 후 FAISS 인덱스 파일(/app/faiss_db/) 유실 발생
- 인덱스 없이 /rag/chat 호출 시 500 에러
- 에러: "FileNotFoundError: faiss_db/index.faiss not found"

----------------------------------------------------------------
2. 원인
----------------------------------------------------------------
- Azure VM B2s 플랜 임시 디스크에 faiss_db/ 저장됨
- 임시 디스크는 VM 재시작 시 초기화됨
- 영구 디스크에 저장하는 로직 미구현 상태

----------------------------------------------------------------
3. 해결
----------------------------------------------------------------
[2026-05-27 10:00] 이성우
- faiss_db/ 저장 경로를 Azure 영구 데이터 디스크로 변경
- .env: FAISS_DIR=/mnt/data/faiss_db
- embedder.py: os.makedirs(FAISS_DIR, exist_ok=True) 추가

[2026-05-27 14:00] 김예은
- Azure 영구 데이터 디스크 마운트 완료 (/mnt/data)
- 권한 설정: chmod 755 /mnt/data/faiss_db

[2026-05-28 09:00] 이성우
- VM 강제 재시작 후 FAISS 인덱스 유지 확인 ✅
- /rag/chat 정상 동작 확인 ✅
- ISSUE-006 해결 완료

----------------------------------------------------------------
4. 재발 방지
----------------------------------------------------------------
- 인덱스 파일 일 1회 PostgreSQL DB에 메타데이터 백업
- FAISS_INDEXES 테이블에 index_path, status 기록
- 인덱스 손상 감지 시 자동 재생성 로직 추가 검토
================================================================

이슈 로그 — /chat 엔드포인트 project_id 누락
================================================================
이슈 ID: ISSUE-007
제목: AI Assistant project_id 미전달로 전체 문서 검색 문제
심각도: High
상태: Resolved
발생일: 2026-05-28
해결일: 2026-05-29
담당자: 이성우, 박주원
보고자: 박주원

----------------------------------------------------------------
1. 증상
----------------------------------------------------------------
- AI Assistant 질문 시 다른 프로젝트 문서 내용이 답변에 포함됨
- 예: OpsRadar 질문인데 다른 팀 회의록 내용이 섞임
- project_id 없이 전체 FAISS 인덱스 검색하는 버그

----------------------------------------------------------------
2. 원인
----------------------------------------------------------------
- 프론트 sendMsg 함수에서 project_id를 요청 바디에 미포함
- 백엔드 /rag/chat에서 project_id 없으면 전체 인덱스 검색

----------------------------------------------------------------
3. 해결
----------------------------------------------------------------
[2026-05-28 15:00] 박주원
- patchChatActions 함수 수정
- document.querySelector('[data-project-id]') 로 project_id 동적 조회
- 없으면 폴백: "30000000-0000-0000-0000-000000000001"

[2026-05-29 09:00] 이성우
- /rag/chat 백엔드: project_id 필수 파라미터로 변경
- project_id 기준으로 해당 프로젝트 FAISS 인덱스만 검색

[2026-05-29 10:00] 검증 완료
- OpsRadar 프로젝트 문서만 검색됨 ✅
- ISSUE-007 해결 완료
