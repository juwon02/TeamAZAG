# Project AZAG Final Frontend Structure

작성 기준: 2026-05-18

이 문서는 현재 거의 최종으로 보는 프론트 구성을 보존하기 위한 화면 구조 메모입니다.

## 1. 대시보드

- AI Weekly / Monthly Summary
- 프로젝트 보드
- 팀원 업무 현황
- 자료실

## 2. AI 분석

- 자료 파일 업로드
- 파일명별 분석 상태 표시
- 상태값: 완료, 분석중
- 추후 메일/채팅 연동 가능하도록 자료 소스 타입 분리

## 3. TODO 관리

- 진행 Todo
- AI 제안
- 완료
- 업무, 담당자, 상태, 액션, 마감일 표시
- 제안 TODO 목록에 대한 근거 문서 표시
- 추적 불가능하거나 누락되는 업무 기록은 추가 고민 필요
- 현재 처리 방향:
  - AI가 잡은 TODO는 근거 문서와 근거 날짜를 표시
  - 근거가 약한 TODO는 담당자 수기 등록 또는 확인 요청 상태로 유지

## 4. 이슈 로그

- 마감일 3일 전에는 이슈로 뜨도록 날짜 기준 자동 노출
- 자연어를 파악해서 AI가 이슈라고 감지할 수 있는지 테스트 필요
- 예시 자연어:
  - 늦을 것 같다
  - 막혔다
  - 스펙이 불명확하다
  - 일정이 위험하다

## 5. 보고서 생성

- 주간 보고
- 월간 보고
- AI 초안 생성 후 사람이 최종 수정
- 근거 부족 항목은 표시 필요

## 6. 지식 전달

- 신규 입사자 온보딩 탭
- 재직자 인수인계 탭
- 권한에 따라 볼 수 있는 문서만 노출하는 방향

## 7. AI Assistant

- 기존 화면을 거의 유지
- 자료실, TODO, 이슈 로그의 근거 문서와 날짜를 기반으로 답변
- 빠른 질문:
  - 가장 위험한 이슈
  - 마감 임박 업무
  - 근거 부족 TODO
